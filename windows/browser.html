Chrome 146 · Web APIs
Chrome's HTML-in-Canvas API
is here — and it's wild
Google's experimental layoutsubtree attribute lets browsers render live DOM elements directly into a WebGL texture — enabling a new class of spatial UIs.
Read More
Deep Dive
What does it mean for the web?
For years, developers have hacked around the browser's compositing model to get HTML content into WebGL. Libraries like html2canvas used screenshot tricks, while CSS3DRenderer faked depth with matrix transforms. Neither approach gave you true WebGL integration.
The new layoutsubtree canvas attribute changes that. Child DOM nodes are rasterized by the browser's own layout engine and exposed to the canvas paint event — meaning hover states, CSS animations, and interactive elements all work inside the texture.
